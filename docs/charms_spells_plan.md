# План: Заклинания и Чармы (милстоуны M6–M7)

Документ описывает, КАК устроены две новые системы и в каком порядке их собирать.
Числа черновые — подберём «на ощупь» при тесте. Каждый под-шаг должен быть играбелен.

---

## M6 — Заклинание «Vengeful Spirit» (мстительный дух)

**Идея (как в Hollow Knight):** нажал клавишу → потратил **33 soul** → перед игроком
вылетает снаряд-дух, летит по горизонтали, бьёт первого врага и исчезает.

### Что переиспользуем (уже готово в коде)
- `GameState.can_cast()` / `spend_soul_for_cast()` — проверка и трата 33 soul.
- Урон врагам: группа `"enemy"` + метод `take_damage(amount)`.
- `player._facing` — куда смотрит игрок (направление полёта снаряда).

### Что добавляем
1. **Действие ввода** `cast` в Input Map (project.godot). Предложение: клавиша **I**
   (рядом с движением правой рукой; J=атака, K=даш, L=лечение, I — над ними).
2. **Сцена снаряда** `scenes/objects/spell_projectile.tscn` + `spell_projectile.gd`:
   - Корень — `Area2D` (он умеет ловить пересечения, физика не нужна — летит по прямой).
   - Внутри: `Polygon2D` (заглушка-визуал, цветной ромб) + `CollisionShape2D`.
   - Скрипт: летит в заданную сторону (`speed ~600`), живёт `lifetime ~0.8с`,
     при касании врага → `take_damage(spell_damage ~15)` и `queue_free()`.
     Маска коллизии — слой 3 (enemy), как у хитбокса гвоздя.
3. **Каст в player.gd** — метод `_try_cast()`:
   - по нажатию `cast`, если `GameState.spend_soul_for_cast()` вернул true →
     `instantiate()` снаряда, задать ему направление = `_facing`, поставить позицию
     чуть впереди игрока, добавить в дерево сцены (`get_parent().add_child(...)`).
   - Это новый для нас приём: **создание сцены в рантайме** (а не заранее в редакторе).
4. **Открытие способности (gating).** Два варианта — выбрать при реализации:
   - (a) доступно сразу (проще тестировать);
   - (b) как даш — флаг `GameState.has_spell`, открывается пикапом (метроидвания-стиль).
   Рекомендация: сделать флаг `has_spell`, но в M6 включить его по умолчанию `true`,
   а пикап добавить позже одной строкой.
5. **HUD/доки:** добавить подсказку «Заклинание: I», обновить `game_design.md` и CLAUDE.md.

**Проверка M6:** SOUL ≥ 33 → нажать I → ромб летит вперёд, бьёт мишень, soul −33.
При soul < 33 — ничего не происходит.

### Задел на будущее (НЕ в M6, просто чтобы архитектура не мешала)
- Второй спелл «вверх» (Howling Wraiths) и «вниз» (Desolate Dive) — та же сцена-снаряд,
  другое направление/форма. Поле `direction: Vector2` в снаряде делает это тривиальным.

## M7 — Чармы (амулеты)

**Идея (как в Hollow Knight):** есть набор амулетов. У игрока ограниченное число
**ячеек (notches)** — например, 3. Каждый амулет занимает 1–3 ячейки. Экипируешь
комбинацию в пределах ячеек, амулеты меняют характеристики (урон, скорость, soul…).

Систему собираем в 3 под-шага, каждый играбелен.

### M7a — Данные чармов + хранение + применение эффектов (без UI)
1. **Тип-ресурс `Charm`** — новый файл `scripts/charm.gd` (`extends Resource`,
   `class_name Charm`). Custom Resource — это «структура данных» Godot, которую можно
   сохранять отдельными файлами `.tres` и редактировать в инспекторе.
   Поля (`@export`): `id: String`, `charm_name: String`, `description: String`,
   `notch_cost: int`, и числовые эффекты (плоские поля — так проще новичку, чем скрипты):
   - `nail_damage_bonus: int` (плюс к урону гвоздя)
   - `soul_per_hit_bonus: int` (плюс к soul за удар)
   - `move_speed_mult: float` (множитель скорости, 1.0 = без изменений)
   - `focus_time_mult: float` (множитель времени лечения, <1 = быстрее)
   - `nail_range_mult: float` (множитель дальности хитбокса)
2. **Конкретные чармы** — стартовый набор (как `.tres`-файлы в `assets/charms/`
   или заданы в коде; решим при реализации). Предложение — 5 штук:
   | Чарм | Ячейки | Эффект |
   |---|---|---|
   | Тяжёлый гвоздь | 2 | `nail_damage_bonus = +3` |
   | Длинный гвоздь | 1 | `nail_range_mult = 1.4` |
   | Собиратель душ | 1 | `soul_per_hit_bonus = +5` |
   | Прыткие ноги | 1 | `move_speed_mult = 1.25` |
   | Быстрый фокус | 3 | `focus_time_mult = 0.6` |
3. **Хранилище в GameState:**
   - `notch_capacity: int = 3` — сколько ячеек всего.
   - `owned_charm_ids: Array[String]` — какие амулеты получены.
   - `equipped_charm_ids: Array[String]` — какие сейчас надеты.
   - функции `can_equip(charm)`, `equip(charm)`, `unequip(charm)` (проверяют ячейки).
   - сигнал `charms_changed` (HUD/player пересчитают эффекты).
4. **Применение эффектов.** GameState считает суммарные бонусы и хранит их как простые
   переменные (пересчёт при `charms_changed`): `nail_damage_bonus`, `move_speed_mult` и т.д.
   `player.gd` читает их в точке использования:
   - скорость = `speed * GameState.move_speed_mult`
   - урон гвоздя = `nail_damage + GameState.nail_damage_bonus`
   - soul за удар = `SOUL_PER_HIT + GameState.soul_per_hit_bonus`
   - время фокуса = `focus_time * GameState.focus_time_mult`
   - смещение хитбокса = базовое × `GameState.nail_range_mult`
   (Сначала проверим эффекты, выдав себе все чармы в коде — UI ещё нет.)

**Проверка M7a:** временно экипировать чарм из кода → урон/скорость реально меняются.

### M7b — Экран экипировки (UI)
1. Новая сцена `scenes/ui/charm_menu.tscn` + скрипт (CanvasLayer, поверх игры).
   Список владеемых чармов, выделение, надетые отмечены. Снизу — счётчик ячеек «2/3».
2. Открытие меню: вариант (a) отдельная клавиша (напр. **C**), ставит игру на паузу
   (`get_tree().paused = true`); вариант (b) только у скамейки (как в HK). Рекомендация:
   начать с клавиши **C** (проще тестировать), позже привязать к скамейке.
3. Навигация с клавиатуры: ↑/↓ выбор, Enter — надеть/снять (если хватает ячеек).
   Эмитит `charms_changed`, эффекты применяются сразу.

**Проверка M7b:** открыть меню, надевать/снимать амулеты, видеть смену статов в бою.

### M7c — Получение чармов в мире + сохранение
1. Пикапы чармов в уровне — по образцу `ability_pickup.gd`: при касании добавляют id
   в `owned_charm_ids`, показывают сообщение, исчезают навсегда (паттерн «исчезает,
   если уже получен» — как у пикапа даша).
2. **Сохранение:** в `save_game`/`load_game` (JSON) добавить `notch_capacity`,
   `owned_charm_ids`, `equipped_charm_ids`. После загрузки — пересчитать бонусы.

**Проверка M7c:** подобрать чарм → выйти/зайти (перезапуск) → чарм на месте и надет.

## Порядок и зависимости
1. **M6** (заклинание) — независим, делаем первым (быстрый результат).
2. **M7a → M7b → M7c** — строго по очереди (UI опирается на данные, сейв на UI).

## Принятые решения (зафиксировано 2026-06-25)
- Спелл: флаг `has_spell`, по умолчанию **включён** (пикап добавим позже).
- Меню чармов: **клавиша C** + пауза (привязку к скамейке оставили на потом).
- Чармы: заданы **в коде** (`GameState._build_charm_catalog`), без `.tres`.

План выполнен полностью. Возможные продолжения — в разделе «Задел на будущее» выше
(спеллы вверх/вниз, рост числа ячеек, привязка меню чармов к скамейке).
